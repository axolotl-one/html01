Instalación y Configuración en Windows

  1. Abrir la terminal con windows+r cmd o en PowerShell
  2. Verificar que Git este instalado en la terminal ingresando el comando git --version
  3. De marcar error, continuar con el proceso de instalación
  4. Descargar Git desde su página oficial https://git-scm.com/downloads
    5. Enlace de Descarga
  5. Ejecutar Git y habilitar permisos y realizar ajustes recomendados en la configuración de instalación:
    1. En la sección de Select Components habilitar: Add a Git Bash Porfile to Windows Termnal
    2. En la sección de Choosing the default editor used by Git seleccionar VisualStudioCode as Git's default
  6. Abrir la terminal de PowerShell o CMD
  7. Ejecutar el comando git para visualizar que Git se ejecute correctamente, de lo contrario abrir la terminal de bash
  8. Configurar el nombre de usuario: Ejecutar el comando git config --global user.name "Nombre de Usuario"
  9. Para corroborar la configuración del nombre de usuario, ejecutar el comando git config --list que mostrará el nombre de usuario al final de la lista
  10. Configurar el nombre de usuario: Ejecutar el comando git config --global user.email "ejemplo@ejemplo.com"
  11. Ejecutar nuevamente el comando git config --list para corroborar la configuración del email
  12. La configuración esta lista para usar Git

Comandos

Navegación Commits Ramas Repo Remotos
    Nombre del ComandoDescripción
  1. git --version
    		2. Muestra la versión de Git
  2. git status
    		3. Muestra el estado del commit (sin repo, libre, con cambios pendientes)
  3. git init
    		4. Crea un nuevo repositorio
  4. git diff
    		5. Muestra la comparativa de los cambios realizados desde el commit actual
  5. git add -A
    		6. Anexa todos los cambios pendientes (nuevo, modificado o eliminado) al Staging
  6. git add text*
    		7. Anexa todos los cambios que empiezan con los caracteres antes de * al Staging
  7. git add *.ext
    		8. Anexa todos los cambios que terminan con los caracteres despues de * al Staging
  8. git add ./RutaArchivo
    		9. Anexa los cambios de una carpeta o archivo referenciado por su nombre al Staging
  9. git reset ./RutaArchivo
    		10. Descarta los cambios de una carpeta o archivo referenciado por su nombre del Staging
  10. git reset --hard NameCommit
    		11. Se redirige la cabeza al commit ingresado y borra los cambios no guardados
  11. git commit -m "etiqueta"
    		12. Crea un nuevo commit (version) en la rama actual, etiqueta forzosa
  12. git log
    		13. Enlista todos los commits de la linea temporal (id, etiqueta, usuario, fecha)
  13. git log --oneline
    		14. Enlista todos los commits de la linea temporal (hash, etiqueta)
  14. git show
    		15. Muestra información del último commit creado o del commit actual
  15. git checkout hash
    		16. Visualiza la version del hash ingresado, util para revisiones
  16. git checkout NombreRama
    		17. Direcciona la rama
  17. git checkout -b NombreRama
    		18. Direcciona la rama, si no existe, la crea
  18. git switch --detach
    		19. Hash direcciona el puntero HEAD a la version con el hash ingresado
  19. git switch -c Nombre
    		20. Crea una nueva rama a partir de la version a la que apunta el HEAD
  20. git branch
    		21. Muestra la lista de ramales
  21. git branch -D NombreRama
    		22. Elimina la rama con el nombre ingresado
  22. git cherry-pick hash
    		23. Anexa en la version actual (HEAD) los cambios de la version del hash
    En conflicto por sobreestricuta se puede: Conservar cambios de la version pasada, de la actual o ambos
  23. git merge
    		24. Anexa en la version actual los cambios de un ramal y extrae su historial de versiones
    En conflicto por sobreestricuta se puede: Conservar cambios de la version pasada, de la actual o ambos
  24. git clone https://link-github
    		25. Clona un repositorio remoto al local en una nueva carpeta del mismo nombre
  25. git remote add aliasRepo https://link-github
    		26. Anexa un repositorio remoto de github a git y lo nombra
  26. git remote get-url aliasRepo
    		27. Obtiene la url del repositorio remoto ingresando su nombre
  27. git remote -v
    		28. Obtiene el nombre y url de un repositorio remoto existente
  28. git rm NombreArchivo
    		29. Borra un archivo del Sistema de Archivos y del Área de Archivos
  29. git rm --cached NameFile
    		30. Excluye el archivo del área de preparación para el próximo commit
  30. git push aliasRepo master
    		31. Envia y sube las versiones de la rama master al repositorio remoto
  31. git push aliasRepo ramaLocal:ramaRemota
    		32. Enivia las versiones de la rama local a una remota nueva o existente
  32. git pull aliasRepo
    		33. Actualiza el repositorio local con los cambios de otros colaboradores
  33. git fetch aliasRepo
    		34. Descarga los cambios de otros colaboradores para revisarlos antes de aplicarlos

Glosario

  1. Commit: Es el espacio que almacena los cambios, actua como un punto de guardado, todos los commit conforman el historial de versiones
  2. Staging: Es el espacio donde se preparan los cambios de los archivos y carpetas que se desean guardar en un commit
