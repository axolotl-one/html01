Instrucciones de SQL para Definición de Datos

CREATE Crea bases de datos, tablas, vistas, etceterá

ALTER Agrega o quita campos a una tabla. También modifica los campos como el tipo de dato o tamaño

DROP Elimina un objeto o tabla de una base de datos

TRUNCATE Elimina todos los registros de una tabla sin borrar sus campos

Instrucciones de SQL para Manipulación de Datos (DML)(CRUDs)

INSERT Agrega registros a una tabla

UPDATE Actualiza los cambios en registros existentes

DELETE Borra registros de una tabla

SELECT Busca o consulta información de una base de datos

Instrucciones para Transactions

BEGIN TRANSACTION Inicia una transacción

COMMIT Indica que la transacción fue completada con éxito

ROLLBACK Indica un fallo en la transacción y debe restablecer la Base de Datos

Instrucciones Complementarias de Consultas SQL

INNER JOIN une la información de campos diferentes en una tabla de consulta

LIKE compara los caracteres mostrando solo las coincidencias, consume tiempo

AS crea alias en consultas: Tabla01 AS NuevoNombre

SELECT DISTINCT eliminan los registros duplicados de las columnas de una tabla seleccionadas

Estructura DISTINCT 
SELECT DISTINCT campo1, campo2 ... campoN FROM NombreTabla
Ejemplo de DISTINCT 
SELECT DISTINCT [Orden Detalles], ProductoID, Productos, Producto Nombre FROM [Orden Detalles]
INNER JOIN Ordenes ON [Orden Detalles], OrdenID = Ordenes.OrdenID
INNER JOIN Productos ON [Orden Detalles], ProductoID = Productos.ProductoID
WHERE OrdenFecha >= '04-07-1996'
AND OrdenFecha <= '05-07-1996'
ORDER BY [Orden Detalles].ProductoID
-- Esto hará una consulta sobre los registros de productos sin repeticiones

Subconsultas con WHERE

INTO es una instrucción que genera una tabla nueva a la Base de Datos a partir de los datos generados de la tabla de referencia y no interactua con los datos de la tabla original evitando alteraciones al ser una copia. Si ya existe, se elimina para actualizar

Estructura con INTO 
IF OBJECT_ID (TablaCopiaInto) IS NOT NULL
BEGIN
     DROP TABLE TablaCopiaInto
     -- Borra la tabla copia de INTO si existe
     -- Evita un error al volver a crear la tabla
END
 
SELECT TablaCopiaInto
INTO TablaCopiaInto
FROM TablaOriginal

GROUP BY agrupa un conjunto de registros de una consulta

Estructura con GROUP BY 
SELECT campo1, campo3 FROM tabla01
INNER JOIN tabla02 ON tabla01.campo01 = tabla04.campo06
GROUP BY campo01 --campo a agrupar-- ORDER BY campo03
Ejemplo con GROUP BY 
Region, Categoria, SUM(OrdenDetalles.PrecioUnitario*OrdenDetalles.Cantidad) AS total
FROM Regiones, Territorio, EmpleadoTerritorio, Empleados, Ordenes, OrdenDetalles, Productos, Categorias
-- Sustituye INNER JOIN por FROM tabla01, tabla02, ..., tablaN
WHERE Regiones.RegionID = Territorios.RegionID
AND Territorios.TerritorioID = EmpleadoTerritorio.TerritorioID
AND EmpleadoTerritorio.EmpleadoID = Empleados.EmpleadoID
AND Empleados.EmpleadoID = Ordenes.EmpleadoID
AND Ordenes.OrdenID = OrdenDetalles.OrdenID
AND OrdenDetalles.ProductoID = Productos.ProductoID
AND Productos.CategoriaID = Categorias.CategoriaID
GROUP BY Region, Categoria ORDER BY Region

-- Ejemplo de una función para calcular el IVA
CREATE FUNCTION CalculoIVA(@Price money) RETURNS money
AS
BEGIN
     DECLARE @IVA money

     RETURN @IVA
END

Ejemplo de un TRIGGER 
-- TRIGGER para actualizar los registros de la tabla Productos después de una inserción a la tabla OrdenDetalle
CREATE TRIGGER ActualizarAlamcen
ON [TablaOrdenDetalles]
FOR INSERT --Se ejecutará el TRIGGER antes de la inserción
AS
BEGIN
     DECLARE @CantidadIngresada smallint, @ProductoID int, @CantidadExistente smallint
     SELECT @CantidadIngresada = Unidades, @ProductoID = ProductoID FROM inserted
     SELECT @CantidadExistente = UnidadesEnAlmacen FROM TablaProductos WHERE ProductoID = @ProductoID
     UPDATE TablaProductos SET UnitsInStock = @CantidadExistente - @CantidadIngresada WHERE ProductoID = @ProductoID
END
-- Cambiar CREATE a ALTER para modificar el TRIGGER después de crearlo
-- Ejemplo de Consula: INSERT INTO [TablaOrdenDetalles](OrdenID, ProductoID, Unidades) VALUES(3452, 6415, 3)

Ejemplo de un Procedimiento Almacenado 
CREATE PROCEDURE ActualizarCliente @ClienteID nchar(5), @Compania nvarchar(40), @ReturnID nchar(5) OUTPUT
AS
BEGIN
     INSERT INTO Clientes(ClienteID, Compania) VALUES (@ClienteID, @Compania)
     IF(@@ERROR <> 0)
          BEGIN
               UPDATE Clientes
               SET Clientes, ClienteID = @ClienteID
               Clientes, Compania = @Compania
               WHERE Clientes, ClienteID = @ClienteID
               PRINT('Se realizó un UPDATE')
          END
     ELSE
          BEGIN
               PRINT('Se realizó un INSERT')
          END
     SELECT @ReturnID = ClienteID FROM Clientes WHERE Clientes. ClienteID = @ClienteID
END
-- Consulta de Ejecución: DECLARE @retorno nchar(5) \n EXECUTE AgregarCliente '00021','PUZZLE',@retorno OUTPUT \n SELECT @retorno

Esquemas

Esquema EstrellaTodas las tablas (dimensiones) recaen en una tabla "hechos" coformada por las llaves primarias de las tablas. Permite consultas directas desde tabla "hechos" (N1)

Esquema Copo de NieveSe deriva del Esquema Estrella, aquí las dimensiones en este esquema permiten normalizarse a la tercera forma permitiendo ahorrar memoria, baja la agilidad de consultas

Esquema ConstelaciónSe deriva del Esquema Estrella, aqui las tablas de hechos son multiples, lo que facilita las consultas concretas. Se complica el mantenimiento a medida que crece la Base

Datawarehouse (almacen de datos)

Oriendatado al analisis de información

Orientado al tema, variante en el tiempo, no volatil (solo de lectura)

Util para realizar consultas informes sin alterar la base de datos principal

Principio de separacion de los datos de DB y DWH (ETL)

Extracción: Obtener la información deseada de la BD

Tranformación: Adecuarlos a los esquemas de DWH

Carga: Almacenar los datos en el DWH

Datas Marts

Se alimentan del DWH donde cada una se encarga de un area especifica como un departamento y da acceso limitado al personal designado

Glosario

Procedimientos Almacenados

conjunto de instrucciones que son ejecutadas, pueden recibir parámetros de entrada y generar o no resultados, dentro de ellas se pueden almacenar otros procedimientos almacenadoa

regresan un valor de estado para indicar si la operación fue concluida con exito

Los usuarios pueden ejecutar procedimientos almacenados para tareas especificas sin necesidad de meterse con el código SQL, son operaciones más seguras y permite reutilizar código que fácilita su mantenimiento

Son llamados con la instrucción EXECUTE, a diferencia de las funciones que son llamadas desde instrucciones DML (SELECT, UPDATE, DELETE), limita el acceso a usuarios a la Base de Datos

Criterios ACID: Atomicidad, Consistencia, Aislamiento, Durabilidad:

1. Realiza completamente la transacción o se cancela, 2. Ser finito, 3. Una operación no puede afectar a otras, 4. Persitencia de la operación realizada
